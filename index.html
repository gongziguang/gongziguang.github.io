前端面试小结

2016-09-26
前端面试

面试了几家公司，去了之后一般会先笔试，出的题目虽然不同，但是考察的知识点却相似，趁着记忆还在，把它们梳理总结一下：

去重

去重，除去重复元素，一般会考察数组和字符串去重

首先，数组去重：

方法一：

Array.prototype.unique1 = function(){
    var temp = [];
    for(var i=0,len=arr.length;i<len;i++){
        if(temp.indexOf(arr[i])===-1){
            temp.push(arr[i]);
        }
    }
    return temp;
}

这种去重方式，简单容易想到，但是因为每次都用到indexOf，效率比较差，所以需要优化

方法二：

Array.prototype.unique2 = function(){
    arr.sort();
    var temp[0] = arr[0];
    for(var i=1,len=arr.length;i<len;i++){
        if(arr[i] !== temp[temp.length-1]){
            temp.push(arr[i]);
        }
    }
}

字符串去重：

方法一：字符串转成数组，然后用数组去重，最后将数组合并层字符串
用到的字符串方法：string.split(“”)，array.join(“”);

方法二：正则表达式
“example”.replace(/(.)(?=*\1)/g,””)

##统计重复数最多的元素及个数

function numberOfRepeat(obj){
    //对数组元素计数
    var s={};
    for(var i=0,len=obj.length;i<len;i++){
        if(s.hasOwnProperty(obj[i])){
            s[obj[i]] +=1;            
        }else{
            s[obj[i]] = 1;
        }
    }
    //将对象转成二维数组
    var arr = [];
    for(x in s){
        arr.push([x,s[x]])
    }
    //对二维数组排序
    arr.sort(function(a,b){
        return b[1]-a[1];
    })
    return arr[0]
}

##类型判断

  • typeOf()
    • 可识别标准类型（null除外），number、string、function、undefined
    • 不能识别具体的对象类型（function除外）
  • Object.prototype.toString.call()
    • 一般封装成函数
      function type(obj){return Object.prototype.toString.call(obj).slice(8,-1);}
  • instanceof
    • 可判别内置对象类型
    • 可判别自定义类型
    • 不能判别原始类型
  • constructor
    • 可判别标准类型（undefined，null除外）
    • 可判别内置对象类型
    • 可判别自定义对象类型

##日期格式化
将日期按“YYYY-MM-DD hh:mm:ss”格式输出

/**
 * [format 日期格式化输出]
 * "YYYY-MM-DD hh:mm:ss"
 * @param  {[Date]} date    日期
 * @param  {[String]} pattern 输出格式
 *
 * @return {[String]}         
 */
function format(date,pattern){
    var padding0=function(num){
        if(num<10){
            num="0"+num
        }
        return num;
    }
    var txt = {
        'YYYY':date.getFullYear(),
        'MM':padding0(date.getMonth()+1),
        'DD':padding0(date.getDate()),
        'hh':padding0(date.getHours()),
        'mm':padding0(date.getMinutes()),
        'ss':padding0(date.getSeconds())
    }
    return pattern.replace(/([a-z])(\1)*/ig,function(match){
        return txt[match]
    })
}

##callee和caller的区别
caller指向的是调用函数本身的那个函数，应用场景：主要用于查看函数本身被哪个函数调用

callee是arguments的一个属性，标识函数对象本身的引用，应用场景：一般用于匿名函数

laravel学习笔记

laravel学习笔记

composer的安装

composer是php包管理工具,类似npm 与 nodejs

  1. composer的安装-> 如何安装composer
  2. 设置中国镜像站，由于中国网络的限制，安装一些包可能速度比较慢

在命令行中执行如下命令：

composer config -g repo.packagist composer https://packagist.phpcomposer.com

3.一些常用命令

composer selfupdate 更新composer到最新版

composer install 安装包

composer update 升级包

laravel安装

方法1：

composer global require “laravel/installer=~1.1”

laravel new projectname

方法2：

composer create-project laravel/laravel –prefer-dist

配置环境

1.打开根目录下.env文件，设置数据库参数

2.设置config/database.php 参数

artisan命令行工具

artisan 是laravel内置的命令行接口

常用命令:

php artisan –version 查看artisan版本

php artisan list 查看所有artisan命令

php artisan help migrate 查看某个命令的帮助

Migration数据迁移

  1. 建立文件 php artisan make:migration create_users_table –create=users
  2. 迁移文件会建立在database/migrations目录下，打开刚建立的文件，增加需要的字段
  3. 执行迁移 php artisan migrate

####回滚迁移
回滚上一次的迁移 php artisan migrate:rollback

回滚所有迁移 php artisan migrate:reset

回滚所有迁移并且再执行一遍 php artisan migrate:refresh

生成 Eloquent 模型

php artisan make:model User

注意：若没有特别指定，系统会默认自动对应【model小写的复数形态】的数据表，例如User会自动对应users表。也可以自定义，方法如下：

class User extends Model{
    protected $table = 'my_users';
}

一些常用数据库操作：

User::all()取出所有数据
$user = User::find('id')根据主键取出一条数据
User::where('username',$username)
$user->save()更新模型
$user->delete()删除模型
$user->touch()更新时间戳

###数据表关联
数据库表可能与其他的表相关联，关系一般为以下几种

  • 一对一
  • 一对多
  • 多对多
  • 远层一对多关联
  • 多态关联
  • 多态的多对多关联

####一对一
在user的模型中定义answer

class User extends Model{
  public function answer(){
    return $this->hasOne('App\Answer');
  }
}

在answer模型中定义user

class Answer extends Model{
  public function user(){
    return $this->belongsTo('App\User');
  }
}

####一对多
一对多的例子如Blog中的评论

class Post extends Model {
    public function comments()
    {
        return $this->hasMany('App\Comment');
    }
}

在评论Comment中定义

class Comment extends Model {

    public function post()
    {
        return $this->belongsTo('App\Post');
    }

}

###多对多
在user中定义模型

class User extends Model {

    public function roles()
    {
        return $this->belongsToMany('App\Role');
    }

}

在role中定义

class Role extends Model {

    public function users()
    {
        return $this->belongsToMany('App\User');
    }

}
